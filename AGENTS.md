# Repository Guidelines

## 回答

- ユーザーへの回答は日本語で簡潔かつ丁寧に行ってください。
- マークダウンを使う場合、過度な強調記法は避けてください。

## 計画

- 修正を始める前に計画をマークダウンファイルで .plans フォルダ配下に日本語で生成してください。
- 計画のファイル名は連番とし、1つ目を 001 始まりとして修正にあった適切なファイル名としてください。
- 具体的なファイル編集をする前に、修正案を提示すること。
- 指示があるまで新しい計画ファイルを作成せず、このセッションで使用する計画ファイルが明示されている場合のみその計画に反映すること。
- このセッションで使用する計画ファイルが未指定の場合は、既存の計画ファイルを勝手に選んで更新せず、どの計画ファイルを使うか確認すること。

## Document

以下の API ドキュメントを参照すること
- https://developer.raindrop.io/v1/authentication

## プロジェクト概要

- このリポジトリは Raindrop API からブックマーク一覧を取得する
  Go 製 CLI です。
- 利用者向けの使い方は `README.md` に記載します。
- ブックマークは登録日時の降順で表示します。
- 出力形式は `yyyy/MM/dd : ブックマークタイトル` です。
- 出力はターミナル幅に収まるよう 1 行で省略します。

## 設定ファイル

- OAuth クライアント情報は `~/.config/rdapi/config.toml` から読みます。
  - `auth.client_id`
  - `auth.client_secret`
  - `auth.redirect_uri`
- token 情報は `config.toml` と同じディレクトリの `secret.toml` に保存します。
  - `auth.access_token`
  - `auth.refresh_token`
- `config.toml` に token が残っているケースは考慮しません。
- `client_secret`、`access_token`、`refresh_token` はソースコード、
  README、計画ファイル、テストデータへ実値を記載しないでください。

## 開発

- Go の標準ライブラリで実装できる範囲では、依存を追加しないでください。
- ファイル編集後は `gofmt` を実行してください。
- 動作確認として `go test ./...` を実行してください。
- ビルド済みバイナリ `rdapi` はコミットしないでください。

## コミット

- コミットタイトルと本文の各行は 72 文字以内にしてください。
- 日本語は 2 文字、英数字と記号は 1 文字として換算してください。
- 本文を書く場合は、修正内容の概要を記載してください。
- 本文で列挙だけを行う場合は、リスト記法 `-` を使ってください。
- 複数行本文を書く場合は、`git commit -F` を使ってください。
